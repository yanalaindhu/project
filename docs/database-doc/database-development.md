# TRIVARNA Database Development

## Overview

The TRIVARNA database is designed to manage user information, onboarding assessments, wellness tracking, AI-generated plans, goal tracking, habit management, journaling, burnout prediction, and adaptive scheduling.

The database follows a centralized architecture where all application modules are connected through the **profiles** table. This allows every feature of the system to access the same user information and maintain data consistency.

---

# Database Architecture

```text
auth.users
      │
      ▼
   profiles
      │
      ├── onboarding_responses
      ├── profile_analysis
      ├── ai_plans
      ├── daily_checkins
      ├── journals
      ├── trivarna_scores
      ├── burnout_predictions
      ├── goals
      ├── habits
      │      └── habit_logs
      └── schedules
             └── schedule_tasks
```

---

# Core User Management

## profiles

### Purpose

The profiles table stores the basic information of each user after authentication.

### Information Stored

* Full Name
* Age
* Occupation
* Account Creation Date

### Why It Is Required

This table acts as the central table of the system. Every module is connected to the user through the profile record.

---

# Onboarding Module

## onboarding_responses

### Purpose

Stores the responses submitted by users during the onboarding process.

### Information Stored

* Life Context
* Emotional State
* Wellbeing Drivers
* Stress Information
* Body Health Information
* Lifestyle Information
* Goals
* Productive Time Window

### Why It Is Required

The onboarding process helps the system understand the user's current condition and lifestyle before generating recommendations.

---

## profile_analysis

### Purpose

Stores the AI-generated analysis of onboarding responses.

### Information Stored

* Mind Score
* Body Score
* Lifestyle Score
* Overall Score
* Burnout Risk
* Strengths
* Risks
* Focus Areas
* AI Coach Summary

### Why It Is Required

This table contains the first assessment generated by the AI system and serves as the foundation for personalization.

---

## ai_plans

### Purpose

Stores the initial personalized plan generated after onboarding.

### Information Stored

* Wake-up Time
* Sleep Target
* Personalized Schedule

### Why It Is Required

The system uses onboarding analysis to generate a customized starting plan for the user.

# Adaptive Scheduling Module

## schedules

### Purpose

Stores AI-generated schedules.

### Information Stored

* Schedule Date
* AI Generation Status
* Creation Date

### Why It Is Required

The schedule acts as the daily action plan generated for the user.

---

## schedule_tasks

### Purpose

Stores individual tasks belonging to a schedule.

### Information Stored

* Task Name
* Category
* Start Time
* End Time
* Status

### Why It Is Required

A schedule can contain multiple tasks. This table stores each task separately.

# Foreign Key Structure

```text
profiles.id ← auth.users.id

onboarding_responses.user_id → profiles.id
profile_analysis.user_id → profiles.id
ai_plans.user_id → profiles.id

daily_checkins.user_id → profiles.id
journals.user_id → profiles.id

trivarna_scores.user_id → profiles.id
burnout_predictions.user_id → profiles.id

goals.user_id → profiles.id
habits.user_id → profiles.id

habit_logs.habit_id → habits.id

schedules.user_id → profiles.id
schedule_tasks.schedule_id → schedules.id
```

# Conclusion

The database structure provides a complete foundation for the TRIVARNA platform. It supports user onboarding, wellness tracking, AI analysis, personalized planning, habit management, goal tracking, burnout prediction, journaling, and adaptive scheduling. The centralized relationship model ensures that all modules are connected to a single user profile, making the system scalable, organized, and easy to maintain.
